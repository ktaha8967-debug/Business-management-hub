## Feature 4: Fix Camera/Microphone Permissions (Windows + Mobile)

### Root cause
- Android `MainActivity.java` has no runtime permission handling
- Capacitor needs the `@capacitor/camera` and `@capacitor/microphone` plugins OR manual runtime permission requests
- The web layer uses `navigator.mediaDevices.getUserMedia()` but on Android this requires native runtime permission grants first

### Files to modify

**`android/app/src/main/java/com/ascentra/command/MainActivity.java`**
- Add Capacitor runtime permission handling:
  ```java
  package com.ascentra.command;

  import android.Manifest;
  import android.content.pm.PackageManager;
  import android.os.Bundle;
  import androidx.core.app.ActivityCompat;
  import androidx.core.content.ContextCompat;
  import com.getcapacitor.BridgeActivity;

  public class MainActivity extends BridgeActivity {
      private static final int PERMISSION_REQUEST_CODE = 100;

      @Override
      protected void onCreate(Bundle savedInstanceState) {
          super.onCreate(savedInstanceState);
          requestMediaPermissions();
      }

      private void requestMediaPermissions() {
          String[] permissions = {
              Manifest.permission.CAMERA,
              Manifest.permission.RECORD_AUDIO
          };

          boolean needRequest = false;
          for (String perm : permissions) {
              if (ContextCompat.checkSelfPermission(this, perm) != PackageManager.PERMISSION_GRANTED) {
                  needRequest = true;
                  break;
              }
          }

          if (needRequest) {
              ActivityCompat.requestPermissions(this, permissions, PERMISSION_REQUEST_CODE);
          }
      }
  }
  ```

**`capacitor.config.json`**
- Add Android-specific permissions configuration:
  ```json
  {
    "appId": "com.ascentra.command",
    "appName": "Ascentra Command",
    "webDir": "public",
    "bundledWebRuntime": false,
    "server": {
      "url": "https://taha.mayfairmarketing.online",
      "cleartext": true
    },
    "plugins": {
      "Camera": {
        "androidPermissions": ["android.permission.CAMERA", "android.permission.RECORD_AUDIO"]
      }
    }
  }
  ```

**`android/app/src/main/AndroidManifest.xml`**
- Ensure permissions are already declared (they are - verified at lines 38-44)
- Add `WRITE_EXTERNAL_STORAGE` and `READ_EXTERNAL_STORAGE` if needed for file uploads

**Windows (Electron) fix:**
- `main.js` ALREADY has `setPermissionRequestHandler` at line 70-76 with correct permission names (`media`, `audioCapture`, `videoCapture`, `notifications`)
- **Root cause**: `sandbox: true` at line 40 in `webPreferences` restricts media device access in Electron
- **Fix**: Change `sandbox: true` to `sandbox: false` in `main.js` line 40. Keep `contextIsolation: true` for security. The permission handler already grants camera/mic access.

**`main.js`** (line 40)
- Change `sandbox: true` to `sandbox: false`
- No other changes needed - permission handler at lines 70-76 is already correct
